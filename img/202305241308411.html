public class SelectionSort {
    public static void main(String[] args) {
        int[] arr = {5, 2, 8, 3, 1, 6};
        selectionSort(arr);
        System.out.println(Arrays.toString(arr));
    }


    public static void selectionSort(int[] arr) {
        for (int i = 0; i < arr.length - 1; i++) {
            int minIndex = i;
            for (int j = i + 1; j < arr.length; j++) {
                if (arr[j] < arr[minIndex]) {
                    minIndex = j;
                }
            }
            if (minIndex != i) {
                int temp = arr[i];
                arr[i] = arr[minIndex];
                arr[minIndex] = temp;
            }
        }
    }
}

类构造方法作用及特点

它的主要作用是在对象创建时对其进行初始化，为对象的实例变量赋初值，以便对象能够正确地使用这些实例变量。

类构造方法的特点如下：

  1. 类构造方法的名称必须与类名相同，且没有返回值类型（包括void）。

  2. 类构造方法可以有参数，也可以没有参数。

  3. 类构造方法在对象创建时自动调用，且只会被调用一次。

  4. 如果没有显式地定义类构造方法，则编译器会自动生成一个默认的构造方法，该构造方法没有参数，且不执行任何操作。

  5. 如果定义了一个或多个类构造方法，则编译器不会再自动生成默认构造方法。

需要注意的是，类构造方法与实例方法的区别在于，类构造方法在对象创建时自动调用，且只会被调用一次；而实例方法需要通过对象来调用，并且可以被调用多次。此外，类构造方法也可以用于执行一些静态初始化操作，例如为静态变量赋初值等。

线程在运行状态中哪些情况会进入阻塞状态?

  1. 等待阻塞：当线程调用了wait()方法，使得线程进入等待状态，直到其他线程调用notify()或notifyAll()方法唤醒该线程时，它才会进入就绪状态。

  2. 同步阻塞：当线程在获取synchronized同步锁时，如果锁被其他线程占用，则该线程会进入同步阻塞状态，直到获取到锁后才会进入就绪状态。

  3. 睡眠阻塞：当线程调用sleep()方法时，线程会进入睡眠阻塞状态，直到指定的时间到达或者被其他线程中断。

  4. IO阻塞：当线程执行IO操作时，如果该操作不能立即完成，则线程会进入IO阻塞状态，直到IO操作完成后才会进入就绪状态。

需要注意的是，以上情况只是线程进入阻塞状态的一些常见情况，实际上还有其他情况也会导致线程进入阻塞状态。此外，在多线程编程中，为了避免死锁等问题，需要合理地设计线程的同步和协作机制。

类成员的访问修饰符有哪些及其作用

类成员的访问修饰符主要有以下四种：

  1. public：表示该成员可以被任何类访问，无访问限制。

    任意位置访问

  2. protected：表示该成员可以被当前类、同一包中的其他类以及其他包中的子类访问，但不能被其他包中的非子类访问。

    非本包的非子类不能访问

  3. default（即不加任何修饰符）：表示该成员可以被当前类、同一包中的其他类访问，但不能被其他包中的类访问。

    同一包中可以访问

  4. private：表示该成员只能被当前类访问，其他类无法访问。

    只能在本类访问

这些访问修饰符的作用如下：

  1. 控制对成员的访问权限，从而保证程序的安全性和稳定性。

  2. 限制对成员的访问，使得程序更易维护和扩展。

  3. 提高代码的可读性和可维护性，使得代码更易于理解和修改。

需要注意的是，访问修饰符并不仅仅用于类成员，还可以用于类、接口、枚举等元素的定义。在使用访问修饰符时，应该根据实际需求选择合适的修饰符，并遵循最小化访问原则，即尽量将成员的访问权限限制在最小范围内，以提高程序的安全性和可维护性。

Java的特性

  1. 简单性：Java语言的语法和特性相对简单，易于学习和使用。

  2. 面向对象：Java是一种面向对象的语言，支持封装、继承和多态等面向对象的特性。

  3. 平台无关性：Java程序可以在不同的操作系统和硬件平台上运行，这是因为Java程序在编译后会生成字节码文件，而字节码可以在任何支持Java虚拟机（JVM）的平台上运行。

  4. 安全性：Java提供了安全性机制，如类加载器、字节码校验器和安全管理器等，可以保护计算机免受恶意代码的攻击。

  5. 高性能：Java虚拟机（JVM）可以优化代码执行，提高程序的性能。

  6. 多线程支持：Java提供了多线程编程的支持，可以实现并发执行和异步编程。

  7. 开源性：Java是开源的，任何人都可以查看和修改Java源代码。

面向对象程序设计的基本思想

  1. 将复杂的问题分解为一个个相对独立、易于理解和修改的对象，每个对象封装了数据和操作数据的方法，从而实现了数据和行为的内聚性。

  2. 通过继承、多态等机制，实现代码的重用和扩展，提高了程序的可维护性和可扩展性。

  3. 通过封装、继承、多态等机制，实现了代码的模块化和抽象化，使得程序更易于理解和修改。

  4. 面向对象编程强调对象之间的交互和消息传递，而不是过程或函数之间的调用，从而实现了更加灵活和松耦合的程序设计。

    // 打印数字金字塔
    public static void printPyramid(int n) {
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= n - i; j++) {
                System.out.print(" ");
            }
            for (int k = 1; k <= 2 * i - 1; k++) {
                System.out.print(i);
            }
            System.out.println();
        }
    }

        int n = ch - 'A' + 1;

            for (int j = 1; j <= n - i; j++) {
                System.out.print(" ");
            }
            for (int k = 1; k <= 2 * i - 1; k++) {
                System.out.print((char)('A' + i - 1));
            }
            System.out.println();
        }
